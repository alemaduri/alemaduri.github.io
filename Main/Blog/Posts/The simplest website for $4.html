Step two. Export

After exporting, you may notice that the files are named stupidly

Because of this, the user will see a billboard in the address bar. The solution to this problem, I see, is to manually change the file names and all links in the code manually. This will have to be done manually. To do this we will use VScode

The algorithm is as follows:

  • For each .html file in the folder, copy the part with the billboard after the name
  • Open search with replacement. Paste %20 there and then the copied line after the file name
  • Replace with nothing
  • Remove the billboard from the file name and move on to the next document

Also, the rubbish needs to be removed from the folder names. It is better not to change the names of the pictures.

After this you need to find the index.html file and delete it

Instead, index.html should be the name of the file you want to open first on your site. You need to rename all occurrences of the current file name to index.htm and then rename the file itself.

To do this, go to the website https://www.urlencoder.org/ru/ and enter the current file name there.

Copy it and replace it with ‚Äúindex‚Äù. There is no need to add %20 this time

The folder name with the name corresponding to the name of the main file should also be changed to index

If you did everything according to the instructions, then the site should work. Open index.html using your browser and check if everything works. If you are confused or have any problems, then you can contact me for help in telegram, I will advise you and edit the article.

Step Three. Uploading the site to hosting

We go to GitHub and create a new public repository without a README.md file called ‚Äúname.github.io‚Äù. This can be done directly on github.com, without a terminal or command line.

We drag the files of our site into the repository so that the index.html file ends up in the root directory.

Let's go to the repository settings. In the left side menu we find ‚ÄúCode and automation‚Äù and ‚ÄúPages‚Äù. For source, select main/(root)

We wait 5-10 minutes for our files to be processed.

Go to your github account settings in the ‚ÄúCode, planning and automation‚Äù column, find Pages and add a domain.

Now go to your personal account on reg.ru > Domains > your domain. Changing DNS servers to ns1.rer.ru and ns2.reg.ru

Add a TXT resource record with values from Github.

Immediately add all other necessary DNS records

We return to Github and click ‚ÄúVerify‚Äù. This step may need to be repeated after a few hours because... DNS records are not updated immediately.

After you have verified your domain, return to the repository with our website and add Custom Domain www.example.ru

If you did everything correctly, then after an hour the ‚ÄúEnforce HTTP‚Äù checkbox should be available.

Done! Now you can go to your website from any browser anywhere in the world.

To update the site, add pages/content to Notion and repeat all the steps in the second step. After that, we upload the new version of the site to the repository, replacing the old one.

This process can be simplified using the git tools, but if you have a basic understanding of the terminal and developer tools, you can figure it out on your own. The purpose of this article is to provide an explanation for dummies.
